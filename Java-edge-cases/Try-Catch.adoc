[.section-overview]
== Overview

• The **finally** block is used for cleanup or logic that should run almost always at the end of a **try** or **catch**.
• <mark class="important">The key idea is that finally runs whether the code completes normally or an exception occurs.</mark>
• It is commonly used to release resources such as streams, database connections, or locks.

[.section-typical]
== Typical cases where finally executes

• Execution leaves **try** normally with no exception.
• An exception is thrown in **try** and is caught by a matching **catch**.
• An exception is thrown in **try** or **catch** and is not handled in that method (it propagates out), but **finally** still runs before the method exits.
• There is a **return**, **break**, or **continue** inside **try** or **catch**; <mark class="important">finally still executes before control actually leaves the method or loop.</mark>

[.section-not-exec]
== Cases where finally may not execute

• [important]#If the JVM terminates abruptly (for example, an explicit System.exit call), the finally block may be skipped.#

• If the executing thread is forcibly stopped in a way that prevents normal completion, finally might not run.
• If the code in **try** or **catch** never finishes (infinite loop, blocking I/O that never returns), the flow never reaches finally.

[.section-example]
== Example code in Java

[source,java]
----
public class FinallyDemo {

    public static void main(String[] args) {
        System.out.println("Case 1: normal flow");
        demoNormal();

        System.out.println("\nCase 2: exception but caught");
        demoExceptionHandled();

        System.out.println("\nCase 3: exception not caught");
        try {
            demoExceptionUnhandled();
        } catch (RuntimeException e) {
            System.out.println("Caught in main: " + e.getMessage());
        }

        System.out.println("\nCase 4: return in try");
        System.out.println("demoReturn(): " + demoReturn());
    }

    static void demoNormal() {
        try {
            System.out.println("  In try (normal)");
        } finally {
            System.out.println("  In finally");
        }
    }

    static void demoExceptionHandled() {
        try {
            System.out.println("  In try (about to throw)");
            throw new RuntimeException("oops");
        } catch (Exception e) {
            System.out.println("  In catch: " + e.getMessage());
        } finally {
            System.out.println("  In finally");
        }
    }

    static void demoExceptionUnhandled() {
        try {
            System.out.println("  In try (about to throw)");
            throw new RuntimeException("uncaught");
        } finally {
            System.out.println("  In finally");
        }
    }

    static String demoReturn() {
        try {
            System.out.println("  In try (before return)");
            return "returned from try";
        } finally {
            System.out.println("  In finally (still executes)");
        }
    }
}
----

[.section-summary]
== Summary

• [important]#Under normal conditions (success paths, handled exceptions, unhandled exceptions, and returns), finally is executed and is a good place for cleanup.#
• In abnormal conditions like abrupt JVM shutdown or non-terminating code, finally is not guaranteed to run, so critical logic should not rely solely on it.
• Use finally mainly for releasing resources and final housekeeping after a try-catch block.
